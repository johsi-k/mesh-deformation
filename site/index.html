Shape and Volume Preserving Rotational Mesh Deformation

Project Summary

Our project implements a research paper that prescribes a method of mesh deformation that preserves both volume and shape. The prescribed method preserves shape by solving a sparse linear system that minimizes the geometric distance integral at every vertex by reducing it to the energy function of a rotation field and minimizing that instead. Solving the linear system yields the angles of rotation between two sets of vertices, where one of the sets is manipulatable by the user. Using these angles, the mesh is reconstructed by solving another linear system which contains constraints on the edges of the mesh to obtain the deformed vertex points. Since there are more edges than vertices, the system is solved in the least squares sense to obtain the closest possible vertex points.

Secondly, the volume of this deformation is preserved by applying a scaling factor on the normals of each moving frame, proportional to the previous depth of the mesh - acting as an approximation for preserving volume.

Implementation

We primarily used C++ and the libraries, Eigen - for linear algebra in C++, Surface_mesh - to perform face and vertex level manipulations, LibIGL - for its curvature scripts as well as linear system solvers and OpenGL - to render geometry.

Introduction

Traditionally, mesh deformations involve solving a linear system constructed between two sets of vertices - a fixed set and a handle set. The fixed set of vertices is the set of vertices whose position remains fixed throughout the deformation, while the handle set is that which can be manipulated by the user. The rest of the vertices not included in either of these sets fall into the interpolated set of vertices, whose positions are calculated by solving the above mentioned linear system. Based on these three sets of vertices, a linear system of equations is constructed, with the positions of the interpolated vertices being the unknowns. The linear system is then solved for the positions of the interpolated set of vertices and the mesh is reconstructed with the new values. This linearly interpolating method of mesh deformation preserves the shape of the mesh to produce a semi-realistic deformation.

Volume and Shape Preservation

The paper introduces two new developments. Using a moving frames representation of meshes, a rotation field is computed which minimizes the energy functional representing the geometric distance between the original and deformed meshes. As the rotation field is applied to the moving frames, an isometry is obtained which preserves surface details and thus the shape of the original mesh. The rotated moving frames are then scaled by a factor proportional to the depth of the original mesh, preserving volume. Together, the developments enable large deformations that preserve both shape and volume to be applied in a single step.

User Interface

The rendering code was adapted from Assignment 0 and Assignment 1. The user is able to select a fixed set of vertices as well as a handle set of vertices and rotate the handle set of vertices about a single axis, in degrees, using their mouses. We cast rays from the position of the camera towards the direction it faced using the camera’s transformations, to implement mouse picking for the various selections above.


Figure 2 : Transformations performed to generate ray in world coordinates from mouse coordinates

The following commands are available,

Press and hold the following keys to,

f - Select fixed vertices by moving mouse over mesh (indicated by green vertices and faces) while holding this button

h - Select handle vertices by moving mouse over mesh (indicated by blue vertices and faces) while holding this button

e - Erase current selection for either fixed or handle vertices by moving mouse over selected parts of mesh while holding this button

q - Perform rotation by using the mouse wheel while holding this button

Figure 3.  Selecting fixed vertices Figure 4. Selecting handle vertices

Press the following keys to,

x - Switch axis of rotation to be around the X axis

y - Switch axis of rotation to be around the Y axis

z - Switch axis of rotation to be around the Z axis

r - Reset any applied mesh deformations and selections

l - Activate an automatic loop mode, where the mesh is rotated between 180 and -180 back and forth

v - Toggle volumetric preservation

w - Toggle between wireframe and solid mesh rendering modes

The mesh as well as vertices placed at each vertex of the mesh (indicated by small solid spheres) are rendered in the viewport.

The vertices and faces take the following colors,

Grey - Ordinary vertices or interpolated vertices

Blue - Set of vertices (along with attached faces) that have been selected to be handle vertices

Green - Set of vertices (along with attached faces) that have been selected to be fixed vertices

Magenta - Set of vertices (along with faces) that the mouse is currently hovering over

We fixed a few memory allocation issues that was causing the mesh to fail when reconstructing since the presentation. We also improved on the UI by shading triangle faces as well as vertices for better visibility of the selected regions.

Results

We are able to deform some meshes while maintaining the relative form. However, volume preservation does not work well, likely because we are not using the principal directions for the moving frames as mentioned in the paper or because of incorrect depth calculation.

Conclusion

Conclusion, learning points,

Further Improvements

  • Currently, we are using the xyz-axes as our moving frame instead of using the principal curvatures at each vertex.
  • Depth calculation involves raycasting which is extremely slow. An acceleration structure could be used to speed it up.
  • Calculation of deformation angles is currently only done for 1 dimension. The paper describes a method for calculating deformation on all 3 angles at once, allowing more versatile deformations.
  • Our program is unable to handle meshes with a large number of vertices (>10,000)
